# Tableau AI Analysis Assistant 产品手册

## 产品概述

Tableau AI Analysis Assistant 是一款基于人工智能的智能数据分析助手，专为 Tableau 用户设计。它允许用户通过自然语言与数据进行交互，自动生成查询、执行分析并提供智能洞察。

### 核心价值

- **降低使用门槛**：无需掌握 VizQL 语法，用自然语言即可查询数据
- **提升分析效率**：AI 自动识别数据模式、异常和趋势
- **智能探索引导**：系统主动推荐后续分析方向
- **企业级可靠性**：完善的重试、校验和容错机制

## 功能特性

### 1. 自然语言查询

用户可以用日常语言提问，系统自动理解并转换为数据查询。

**支持的查询类型**：
- 聚合查询：「各地区的销售额是多少」
- 时间分析：「上个月的订单趋势」
- 对比分析：「今年和去年的销售对比」
- 排名分析：「销售额前10的产品」
- 占比分析：「各类别的销售占比」

**智能时间识别**：
- 相对时间：「上个月」「去年同期」「最近7天」
- 绝对时间：「2024年1月」「2024-01-01到2024-03-31」
- 财务周期：「上个季度」「本财年」

### 2. 智能字段映射

系统自动将业务术语映射到数据源的技术字段。

**映射策略**：
- 高置信度（≥0.9）：直接返回，无需 LLM 确认
- 中置信度（0.7-0.9）：LLM 重排序选择最佳匹配
- 低置信度（<0.7）：LLM 智能推断

**示例**：
| 用户输入 | 映射结果 |
|----------|----------|
| 销售额 | [Sales Amount] |
| 地区 | [Region] |
| 产品类别 | [Product Category] |

### 3. 渐进式洞察分析

采用「AI 宝宝吃饭」模式，分阶段分析数据，避免信息过载。

**Phase 1 - 统计分析**（无 LLM）：
- 数据分布检测（正态、偏态、均匀等）
- 异常值检测（IQR、Z-Score）
- 帕累托分析（80/20 法则）
- 聚类分析

**Phase 2 - LLM 分析**（双 LLM 协作）：
- 分析师 LLM：深入分析数据块
- 主持人 LLM：决定分析方向和停止时机

### 4. 智能重规划

系统自动评估分析完整性，推荐后续探索方向。

**评估维度**：
- 完成度评分（0-1）
- 已回答的问题
- 潜在的探索方向

**推荐类型**：
- 钻取建议：「按城市细分地区销售」
- 汇总建议：「查看整体趋势」
- 对比建议：「与去年同期对比」

### 5. Token 级流式输出

实时推送 LLM 生成的每个 token，提供流畅的用户体验。

**事件类型**：
| 事件 | 说明 |
|------|------|
| `node_start` | 节点开始执行 |
| `token` | LLM 生成的 token |
| `node_complete` | 节点执行完成 |
| `complete` | 整个流程完成 |

## 使用指南

### 快速开始

1. **打开 Tableau 看板**
   - 确保看板已配置 AI Assistant 扩展

2. **输入问题**
   - 在输入框中用自然语言描述你的分析需求
   - 示例：「各产品类别的销售额是多少」

3. **查看结果**
   - 系统会实时显示分析进度
   - 最终结果包含数据和洞察

4. **探索更多**
   - 根据系统推荐继续深入分析
   - 或提出新的问题

### 最佳实践

**提问技巧**：
- ✅ 明确指定度量和维度：「按地区统计销售额」
- ✅ 指定时间范围：「2024年第一季度的订单数」
- ✅ 使用业务术语：「VIP客户的平均客单价」
- ❌ 避免模糊表述：「数据怎么样」
- ❌ 避免过于复杂：一次问一个问题

**常见问题类型**：

| 类型 | 示例 |
|------|------|
| 汇总 | 「总销售额是多少」 |
| 分组 | 「各地区的销售额」 |
| 排名 | 「销售额前10的产品」 |
| 趋势 | 「月度销售趋势」 |
| 对比 | 「今年和去年的对比」 |
| 占比 | 「各类别的销售占比」 |

## 系统架构

### 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Tableau Dashboard                         │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              AI Assistant Extension                  │    │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐             │    │
│  │  │ 输入框  │  │ 结果区  │  │ 推荐区  │             │    │
│  │  └─────────┘  └─────────┘  └─────────┘             │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │ SSE
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    Backend Service                           │
│  ┌─────────────────────────────────────────────────────┐    │
│  │                 LangGraph Workflow                   │    │
│  │  Understanding → FieldMapper → QueryBuilder →        │    │
│  │  Execute → Insight → Replanner                       │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    External Services                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Tableau API │  │   LLM API   │  │   SQLite    │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 工作流节点

| 节点 | 功能 | 类型 |
|------|------|------|
| Understanding | 问题理解和分类 | LLM |
| FieldMapper | 字段语义映射 | RAG + LLM |
| QueryBuilder | VizQL 查询生成 | 代码 |
| Execute | 查询执行 | 代码 |
| Insight | 数据洞察分析 | 双 LLM |
| Replanner | 重规划决策 | LLM |

## 配置说明

### 环境变量

```bash
# Tableau 连接
TABLEAU_DOMAIN=https://your-server.com
TABLEAU_SITE=your-site
DATASOURCE_LUID=your-datasource-luid

# 认证方式（二选一）
# JWT 认证（推荐）
TABLEAU_JWT_CLIENT_ID=xxx
TABLEAU_JWT_SECRET_ID=xxx
TABLEAU_JWT_SECRET=xxx

# PAT 认证
TABLEAU_PAT_NAME=xxx
TABLEAU_PAT_SECRET=xxx

# LLM 配置
LLM_MODEL_PROVIDER=deepseek  # local/openai/deepseek/zhipu
LLM_API_KEY=your-api-key
```

### 性能调优

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SUMMARIZATION_TOKEN_THRESHOLD` | 20000 | 对话历史总结阈值 |
| `MODEL_MAX_RETRIES` | 3 | LLM 重试次数 |
| `MAX_REPLAN_ROUNDS` | 3 | 最大重规划轮数 |
| `METADATA_CACHE_TTL` | 86400 | 元数据缓存时间（秒） |

## 故障排除

### 常见问题

**Q: 查询返回空结果**
- 检查数据源是否有数据
- 确认字段映射是否正确
- 查看时间范围是否合理

**Q: 响应速度慢**
- 检查 LLM API 连接
- 考虑启用预热功能
- 检查网络延迟

**Q: 字段映射错误**
- 使用更明确的业务术语
- 检查数据源字段描述是否完整
- 联系管理员更新字段索引

### 错误代码

| 代码 | 说明 | 解决方案 |
|------|------|----------|
| AUTH_001 | 认证失败 | 检查 JWT/PAT 配置 |
| QUERY_001 | 查询构建失败 | 检查字段映射 |
| LLM_001 | LLM 调用失败 | 检查 API 配置 |
| VIZQL_001 | VizQL 执行失败 | 检查数据源连接 |
