# 字段检索优化需求文档

## 概述
优化 FieldRetriever 组件，使其能够更好地将用户查询术语与数据模型中的实际字段名称进行匹配，特别是当字段名称与用户术语不直接匹配时。

## 问题陈述
当前 FieldRetriever 使用基于 RAG 的语义搜索来查找相关字段。然而，当用户查询使用业务术语（如"销售额"）而实际字段名称不同（如"sale_cost_30"、"netamt_1"）时，检索返回低置信度、语义不相关的结果。

## 用户故事

### 1. 语义字段匹配
**作为** 数据分析师  
**我希望** 系统能理解"销售额"可能映射到"netamt_1"或"revenue"等字段  
**以便** 我可以使用自然的业务术语查询数据

#### 验收标准
- 1.1 搜索业务术语时，系统返回具有相似语义含义的字段
- 1.2 系统在匹配时考虑字段描述、类别和样本值
- 1.3 置信度分数准确反映查询与字段之间的语义相似度

### 2. 字段别名支持
**作为** 系统管理员  
**我希望** 能够为字段定义别名/同义词  
**以便** 用户可以使用常见的业务术语进行查询

#### 验收标准
- 2.1 支持在配置或元数据中定义字段别名
- 2.2 别名被包含在搜索索引中以提高匹配效果
- 2.3 多个别名可以映射到同一个字段

### 3. 查询增强
**作为** 用户  
**我希望** 我的简单查询能够被增强并添加上下文  
**以便** 搜索返回更相关的结果

#### 验收标准
- 3.1 搜索查询使用角色上下文（度量/维度）进行增强
- 3.2 自动添加相关术语以提高召回率
- 3.3 查询增强不会显著影响搜索延迟

### 4. 降级机制
**作为** 用户  
**我希望** 即使找不到精确匹配，系统也能提供合理的结果  
**以便** 我仍然可以继续进行分析工作

#### 验收标准
- 4.1 当 RAG 检索返回低置信度结果时，降级到替代策略
- 4.2 降级策略包括：精确子串匹配、基于类别的过滤、LLM 辅助映射
- 4.3 系统在使用降级结果时明确指示

### 5. 索引质量改进
**作为** 开发者  
**我希望** 字段索引包含丰富的语义信息  
**以便** 基于嵌入的搜索能够有效工作

#### 验收标准
- 5.1 索引文本包含字段标题、描述、类别和样本值
- 5.2 索引文本格式化以最大化嵌入质量
- 5.3 当字段元数据变化时索引自动更新

## 技术约束
- 必须保持与现有 FieldRetriever API 的向后兼容性
- 搜索延迟增加不应超过 50ms
- 必须与现有 RAG 基础设施（FAISS + embeddings）配合工作
- 配置应遵循现有 app.yaml 模式

## 范围外
- 搜索期间的实时字段值查找
- 中英文以外的多语言支持
- 自定义嵌入模型训练
