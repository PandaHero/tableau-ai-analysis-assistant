# Analytics Assistant 前端需求文档

## 1. 项目概述

### 1.1 项目背景

Analytics Assistant 是一个基于 AI 的数据分析助手，通过自然语言理解用户的分析需求，自动生成并执行 Tableau 查询，返回分析结果和业务洞察。

**当前状态**：
- ✅ 后端工作流已完成（语义解析、字段映射、字段语义推断）
- ✅ Tableau GraphQL 集成已完成
- ✅ 流式输出 API 已实现
- ❌ 前端界面尚未开发

**目标**：
- 开发一个嵌入 Tableau 仪表板的扩展插件前端
- 提供流畅的对话式数据分析体验
- 支持实时流式输出，展示 AI 思考过程
- 清晰展示查询结果和业务洞察

### 1.2 技术约束

| 约束项 | 说明 |
|--------|------|
| 部署方式 | Tableau Extension（嵌入式 Web 应用） |
| 后端 API | FastAPI + SSE 流式输出 |
| 数据源 | Tableau 数据源（通过 GraphQL 访问） |
| 浏览器兼容 | Chrome 90+, Edge 90+, Safari 14+, Firefox 88+ |
| Tableau 版本 | Tableau Desktop/Server/Cloud 2021.1+ |

### 1.3 用户角色

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| 数据分析师 | 使用 Tableau 进行日常数据分析 | 快速查询销售数据、生成报表 |
| 业务用户 | 非技术背景，需要数据洞察 | 用自然语言提问，获取业务建议 |
| 管理者 | 需要快速决策支持 | 查看关键指标、趋势分析 |


## 2. 核心需求

### 需求 1: 对话式交互界面

**用户故事**: 作为数据分析师，我希望通过自然语言与 AI 对话，以便快速获取数据分析结果。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 用户打开扩展 THEN 显示欢迎页面，包含示例问题
2. WHEN 用户输入问题并发送 THEN 问题显示在对话区域右侧（用户消息）
3. WHEN AI 开始处理 THEN 显示思考状态指示器
4. WHEN AI 回复 THEN 回复显示在对话区域左侧（AI 消息）
5. WHEN 对话区域有新消息 THEN 自动滚动到最新消息

### 需求 2: 实时流式输出

**用户故事**: 作为用户，我希望看到 AI 实时生成回复，以便了解处理进度。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 后端发送 SSE token 事件 THEN 实时追加显示文字（打字机效果）
2. WHEN 后端发送 thinking 事件 THEN 更新思考状态为对应阶段
3. WHEN 后端发送 complete 事件 THEN 结束流式输出，显示完整内容
4. WHEN 后端发送 error 事件 THEN 显示错误消息
5. WHEN SSE 连接中断 THEN 显示"连接已断开"提示和重连按钮

### 需求 3: Markdown 内容渲染

**用户故事**: 作为用户，我希望 AI 回复支持格式化文本，以便更好地阅读分析结果。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN AI 回复包含 Markdown THEN 正确渲染标题、列表、加粗、斜体
2. WHEN AI 回复包含代码块 THEN 显示语法高亮（支持 json, sql, python）
3. WHEN AI 回复包含表格 THEN 正确渲染 GFM 表格语法
4. WHEN AI 回复包含链接 THEN 链接可点击
5. WHEN 渲染 HTML THEN 转义 XSS 向量（`<script>`, `onerror` 等）


### 需求 4: 数据表格展示

**用户故事**: 作为用户，我希望查询结果以表格形式展示，以便查看详细数据。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 查询返回数据 THEN 在消息中显示数据表格
2. WHEN 数据行数 > 10 THEN 默认显示前 10 行，提供"展开全部"按钮
3. WHEN 表格宽度超出容器 THEN 启用水平滚动
4. WHEN 用户点击表头 THEN 按该列排序（升序→降序→原始顺序）
5. WHEN 用户点击"导出 CSV" THEN 下载 CSV 文件
6. WHEN 显示数值 THEN 整数使用千分位分隔符，小数保留 2 位

### 需求 5: Tableau 数据源集成

**用户故事**: 作为用户，我希望扩展能自动获取当前仪表板的数据源，以便无需手动配置。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 扩展初始化 THEN 调用 Tableau Extensions API 获取当前仪表板的数据源列表（遍历所有 worksheet 并去重）
2. WHEN 获取到数据源 THEN 自动选择第一个数据源
3. WHEN 用户发送问题 THEN 请求中包含数据源名称（datasourceName）
4. WHEN Tableau API 调用失败 THEN 显示错误提示"无法获取数据源，请检查 Tableau 连接"，禁用输入功能
5. WHEN 用户切换数据源 THEN 在设置面板中提供数据源下拉选择器（仅显示当前仪表板的数据源列表）
6. WHEN 数据源列表为空 THEN 显示"当前仪表板没有可用数据源"提示，禁用输入功能
7. WHEN 获取数据源 THEN 只获取当前仪表板使用的数据源，不获取所有已发布的数据源

### 需求 6: 错误处理与用户反馈

**用户故事**: 作为用户，我希望在出错时获得清晰的提示，以便知道如何处理。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 网络请求失败 THEN 显示"网络连接失败，请检查网络后重试"和"重试"按钮
2. WHEN 后端返回 4xx/5xx 错误 THEN 显示用户友好的错误消息
3. WHEN 请求超时（> 60 秒） THEN 显示"分析时间较长，请稍候"和"取消"按钮
4. WHEN 输入为空或仅空白字符 THEN 禁用发送按钮
5. WHEN 输入超过 2000 字符 THEN 显示字符计数器并阻止继续输入

### 需求 7: 多轮对话上下文

**用户故事**: 作为用户，我希望 AI 能理解对话上下文，以便进行连续对话而无需重复背景信息。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 用户发送问题 THEN 请求包含完整对话历史（最近 10 轮）
2. WHEN 用户说"按地区分组" THEN AI 理解是对上一个查询的补充
3. WHEN 用户说"和去年对比" THEN AI 理解需要对比的是之前查询的数据
4. WHEN 对话历史超过 10 轮 THEN 只发送最近 10 轮对话
5. WHEN 用户点击"新对话" THEN 清空上下文，开始新会话

### 需求 8: 消息操作与反馈

**用户故事**: 作为用户，我希望能对 AI 回复进行操作和反馈，以便改进使用体验。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN AI 回复完成 THEN 显示操作按钮（👍 点赞、👎 点踩、📋 复制、🔄 重新生成）
2. WHEN 用户点击👍或👎 THEN 发送反馈到后端（可选填写反馈原因）
3. WHEN 用户点击📋 THEN 复制消息内容到剪贴板，显示"已复制"提示
4. WHEN 用户点击🔄 THEN 使用相同问题重新生成回复
5. WHEN AI 正在回复时 THEN 显示⏹️"停止生成"按钮
6. WHEN 用户点击⏹️ THEN 停止流式输出，保留已生成的部分内容

### 需求 9: 建议问题与快捷回复

**用户故事**: 作为用户，我希望 AI 能建议相关问题，以便继续深入分析而无需思考下一步。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN AI 回复完成 THEN 显示 3 个建议问题按钮
2. WHEN 用户点击建议问题 THEN 自动发送该问题
3. WHEN 建议问题基于当前对话上下文 THEN 问题相关且有价值
4. WHEN 欢迎页面 THEN 显示 5-8 个示例问题
5. WHEN 用户输入时 THEN 显示历史常用问题（可选）

### 需求 10: 数据可视化

**用户故事**: 作为用户，我希望数据能以图表形式展示，以便更直观地理解趋势和对比。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 查询结果适合可视化 THEN AI 自动生成图表配置
2. WHEN 显示图表 THEN 支持折线图、柱状图、饼图、散点图
3. WHEN 用户点击"切换视图" THEN 在表格和图表间切换
4. WHEN 用户与图表交互 THEN 显示详细数据（tooltip、图例）
5. WHEN 用户点击"导出图表" THEN 下载为 PNG 或 SVG
6. WHEN 图表数据过多 THEN 自动聚合或分页显示

### 需求 11: 思考过程可视化

**用户故事**: 作为用户，我希望看到 AI 的思考过程，以便理解分析逻辑和建立信任。

**优先级**: P1（重要功能）

**验收标准**:
1. WHEN AI 处理时 THEN 显示步骤列表（可展开/折叠）
2. WHEN 每个步骤完成 THEN 显示✅和详细信息（如识别的字段、生成的查询）
3. WHEN 步骤进行中 THEN 显示⏳动画
4. WHEN 步骤失败 THEN 显示❌和错误原因
5. WHEN 用户点击步骤 THEN 展开查看详细日志


### 需求 12: 会话管理

**用户故事**: 作为用户，我希望能管理对话历史，以便回顾之前的分析和跨设备访问。

**优先级**: P0（核心功能）

**说明**: 会话数据存储在后端数据库，使用 Tableau 用户身份进行数据隔离

**验收标准**:
1. WHEN 扩展初始化 THEN 自动获取 Tableau 当前用户信息（username）
2. WHEN 用户开始新对话 THEN 自动生成 UUID v4 格式的 Session ID
3. WHEN 用户刷新页面 THEN 从后端 API 恢复当前会话（基于 Tableau 用户）
4. WHEN 用户点击"新对话"按钮 THEN 清空当前对话，生成新 Session ID
5. WHEN 用户切换设备 THEN 可以访问所有历史会话（Tableau 自动认证）
6. WHEN 用户点击"会话历史" THEN 显示该用户的所有历史会话列表（按时间倒序）
7. WHEN 用户点击历史会话 THEN 加载该会话的完整对话记录
8. WHEN 用户点击"删除会话" THEN 从后端删除该会话（需确认）
9. WHEN 用户点击"导出对话" THEN 下载为 Markdown 或 PDF 格式
10. WHEN 会话数据变更 THEN 自动同步到后端数据库
11. WHEN 不同 Tableau 用户使用同一仪表板 THEN 数据完全隔离

### 需求 13: 设置面板

**用户故事**: 作为用户，我希望能配置分析参数，以便个性化使用体验。

**优先级**: P0（核心功能）

**说明**: 用户设置存储在后端数据库，使用 Tableau 用户身份进行数据隔离

**验收标准**:
1. WHEN 用户点击设置按钮 THEN 打开设置面板
2. WHEN 在设置面板 THEN 可以选择数据源（从已获取的数据源列表中选择）
3. WHEN 在设置面板 THEN 可以选择语言（中文/英文）
4. WHEN 在设置面板 THEN 可以选择分析深度（标准/深入分析）
5. WHEN 在设置面板 THEN 可以开启/关闭思考过程展示
6. WHEN 在设置面板 THEN 可以设置默认数据源
7. WHEN 用户修改设置 THEN 自动保存到后端数据库（基于 Tableau 用户）
8. WHEN 用户切换设备 THEN 自动加载用户设置（Tableau 自动认证）
9. WHEN 不同 Tableau 用户使用同一仪表板 THEN 设置完全隔离

### 需求 14: 响应式设计

**用户故事**: 作为用户，我希望界面在不同尺寸下都能正常使用。

**优先级**: P0（核心功能）

**验收标准**:
1. WHEN 窗口宽度 >= 768px THEN 使用标准布局
2. WHEN 窗口宽度 480-768px THEN 使用紧凑布局
3. WHEN 窗口宽度 320-480px THEN 使用最小化布局
4. WHEN 窗口宽度 < 320px THEN 显示"窗口过小"提示
5. WHEN 表格宽度超出容器 THEN 启用水平滚动


## 3. 非功能性需求

### NFR-1: 性能要求

| 指标 | 目标值 | 测量方法 |
|-----|-------|---------|
| 首屏加载时间（FCP） | < 2s | Lighthouse |
| 可交互时间（TTI） | < 3s | Lighthouse |
| 消息渲染延迟 | < 100ms | Performance API |
| 流式输出延迟 | < 50ms | 网络监控 |
| 内存占用（稳定态） | < 100MB | Chrome DevTools |

### NFR-2: 兼容性要求

| 平台 | 最低版本 |
|-----|---------|
| Chrome | 90+ |
| Edge | 90+ |
| Safari | 14+ |
| Firefox | 88+ |
| Tableau Desktop | 2021.1+ |
| Tableau Server | 2021.1+ |
| Tableau Cloud | 当前版本 |

### NFR-3: 安全性要求

| 要求 | 说明 |
|-----|------|
| XSS 防护 | 转义所有用户输入和 AI 输出中的危险字符 |
| HTTPS | 生产环境必须使用 HTTPS |
| API 认证 | 使用 Tableau 用户身份（通过请求头传递） |
| 身份校验 | 后端需校验请求来源/身份，防止前端伪造用户信息 |
| 数据隐私 | 不在前端存储敏感数据（如 API Key） |

### NFR-4: 可维护性要求

| 要求 | 说明 |
|-----|------|
| 代码规范 | 使用 ESLint + Prettier |
| 类型安全 | TypeScript 严格模式 |
| 组件化 | 单一职责原则，组件可复用 |
| 测试覆盖 | 核心功能单元测试覆盖率 > 80% |
| 文档 | 关键组件和工具函数有 JSDoc 注释 |
