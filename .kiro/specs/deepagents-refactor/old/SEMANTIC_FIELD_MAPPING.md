# 语义字段映射设计文档

## 🎯 设计目标

使用 **RAG + LLM** 的混合方案，实现智能的字段映射，替代简单的字符串匹配。

## 🏗️ 架构设计

### 整体流程

```
用户输入："销售额"
  ↓
┌─────────────────────────────────────┐
│ 1. 向量检索（RAG）                   │
│   - 将"销售额"转换为向量              │
│   - 在字段向量库中检索                │
│   - 返回 Top-5 候选字段               │
└──────────────┬──────────────────────┘
               ↓
         候选字段列表
         ["Sales", "Revenue", 
          "Total Sales", "Sales Amount",
          "Sales Quantity"]
               ↓
┌─────────────────────────────────────┐
│ 2. LLM 语义判断                      │
│   - 理解用户问题上下文                │
│   - 分析候选字段的业务含义            │
│   - 选择最佳匹配                     │
│   - 给出置信度和理由                 │
└──────────────┬──────────────────────┘
               ↓
         最终结果
         {
           "matched_field": "Sales",
           "confidence": 0.95,
           "reasoning": "根据上下文..."
         }
```
