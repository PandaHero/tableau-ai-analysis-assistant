# RAG 在本项目中的用途分析

## 🎯 核心问题

**RAG（检索增强生成）在 Tableau Assistant 项目中到底做什么？**

---

## 1. RAG 的三个层次

### 1.1 元数据检索（必须）

**用途**：语义字段映射

**存储内容**：Tableau 元数据（字段信息）
- 字段名、显示名、类型、描述、示例值等
- 约 1000 个字段

**检索场景**：
```
用户说："销售额"
  ↓
向量检索元数据
  ↓
找到：[Sales].[Sales Amount]
```

**数据量**：小（约 1000 个向量）
**更新频率**：低（元数据变化不频繁）
**必要性**：✅ 必须（核心功能）

---

### 1.2 历史查询检索（可选）

**用途**：语义缓存、问题推荐

**存储内容**：历史查询和结果
- 用户问题
- 查询结果
- 洞察分析

**检索场景 1：语义缓存**
```
用户问："2016年各地区的销售额"
  ↓
向量检索历史查询
  ↓
找到相似问题："2016年不同地区的销售金额"
  ↓
相似度 > 0.95
  ↓
直接返回缓存结果（不执行查询）
```

**检索场景 2：问题推荐**
```
用户输入："销售"
  ↓
向量检索历史问题
  ↓
推荐：
  1. "2016年各地区的销售额"
  2. "销售额同比增长"
  3. "各产品类别的销售情况"
```

**数据量**：中（取决于用户数量和查询频率）
**更新频率**：高（每次查询都会新增）
**必要性**：🔮 可选（优化功能）

---

### 1.3 洞察检索（可选）

**用途**：知识库、智能推荐

**存储内容**：历史洞察和分析
- 洞察内容
- 相关字段
- 分析结论

**检索场景 1：相关洞察**
```
用户问："为什么华东利润率低？"
  ↓
向量检索历史洞察
  ↓
找到相关洞察：
  1. "华东地区家具类利润率低"
  2. "华东地区折扣率高"
  ↓
提供参考
```

**检索场景 2：智能推荐**
```
用户查看："华东地区销售额"
  ↓
向量检索相关洞察
  ↓
推荐：
  1. "华东地区利润率分析"
  2. "华东地区产品结构"
```

**数据量**：中（取决于分析次数）
**更新频率**：中（每次分析都会新增）
**必要性**：🔮 可选（高级功能）

## 4. 推荐方案

### 4.1 MVP 阶段（必须）

**只实现元数据检索**：
- 向量数据库：FAISS
- Embedding 模型：bce-embedding-base-v1（有 GPU）或 text-embedding-3-large（无 GPU）
- 存储内容：Tableau 元数据（约 1000 个字段）
- 用途：语义字段映射

**理由**：
- 成本极低
- 核心功能
- 必须实现

---

### 4.2 Phase 2（可选）

**添加历史查询检索**：
- 存储内容：历史查询 + 结果
- 用途：
  1. 语义缓存（避免重复查询）
  2. 问题推荐（帮助用户发现问题）

**理由**：
- 成本可接受
- 提升用户体验
- 减少重复查询

**实施建议**：
```python
class HistoryQueryStore:
    """历史查询存储"""
    
    def __init__(self):
        self.embeddings = SentenceTransformer('maidalun1020/bce-embedding-base-v1')
        self.index = faiss.IndexFlatIP(768)
        self.queries = []
    
    def add_query(self, question, result):
        """添加历史查询"""
        # 1. 向量化问题
        vector = self.embeddings.encode([question])
        vector = vector.astype('float32')
        faiss.normalize_L2(vector)
        
        # 2. 添加到索引
        self.index.add(vector)
        
        # 3. 保存查询和结果
        self.queries.append({
            "question": question,
            "result": result,
            "timestamp": datetime.now()
        })
    
    def search_similar(self, question, threshold=0.95):
        """搜索相似查询"""
        # 1. 向量化查询
        query_vector = self.embeddings.encode([question])
        query_vector = query_vector.astype('float32')
        faiss.normalize_L2(query_vector)
        
        # 2. 搜索
        scores, indices = self.index.search(query_vector, k=1)
        
        # 3. 判断相似度
        if scores[0][0] >= threshold:
            return self.queries[indices[0][0]]
        
        return None
```

## 5. 最终结论

### 5.1 RAG 在本项目中的用途

| 用途 | 存储内容 | 数据量 | 成本 | 优先级 |
|------|---------|--------|------|--------|
| **语义字段映射** | 元数据 | 1000 | 极低 | ✅ 必须 |
| **语义缓存** | 历史查询 | 30,000 | 低 | 🔮 可选 |
| **问题推荐** | 历史查询 | 30,000 | 低 | 🔮 可选 |
| **洞察检索** | 历史洞察 | 150,000 | 中 | 🔮 可选 |

### 5.2 实施路径

**Phase 1（MVP）**：
- ✅ 元数据检索（语义字段映射）
- 成本：¥0（BCE）或 $0.013（OpenAI）

**Phase 2（优化）**：
- 🔮 历史查询检索（语义缓存 + 问题推荐）
- 成本：¥0（BCE）或 $3.9（OpenAI）

**Phase 3（高级）**：
- 🔮 洞察检索（知识库 + 智能推荐）
- 成本：¥0（BCE）或 $19.5（OpenAI）

### 5.3 关键决策

1. **MVP 阶段**：只实现元数据检索
   - 理由：成本极低，核心功能
   - 存储：只存储 Tableau 元数据（约 1000 个字段）

2. **Phase 2**：根据用户反馈决定是否添加历史查询检索
   - 理由：成本可接受，提升用户体验
   - 存储：历史查询 + 结果

3. **Phase 3**：根据需求决定是否添加洞察检索
   - 理由：成本较高，高级功能
   - 存储：历史洞察
