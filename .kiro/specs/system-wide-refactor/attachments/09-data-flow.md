## 三层意图路由流程

### 流程说明

三层意图路由采用**快速失败**策略，从快到慢逐层尝试：

**L0 层（规则引擎）**：
- 速度：< 1ms
- 准确率：95%+（高置信度场景）
- 策略：关键词匹配 + 正则表达式
- 阈值：confidence >= 0.9 → 直接返回

**L1 层（小模型）**：
- 速度：10-50ms
- 准确率：85%+（中等置信度场景）
- 策略：DistilBERT 分类模型
- 阈值：confidence >= 0.8 → 直接返回

**L2 层（大模型）**：
- 速度：500-2000ms
- 准确率：95%+（兜底保证）
- 策略：LLM 推理（GPT-4/Claude）
- 阈值：无阈值，最终结果

### 执行流程

    ↓
┌──────────────────────────────────────┐

    ↓

### 性能对比

| 层级 | 命中率 | 平均延迟 | Token 消耗 | 成本 |
|------|--------|----------|-----------|------|
| L0   | 60%    | <1ms     | 0         | $0   |
| L1   | 30%    | 30ms     | 0         | $0   |
| L2   | 10%    | 1200ms   | ~500      | $0.01 |

**优化效果**：
- 90% 的查询在 L0/L1 完成（延迟 <50ms）
- Token 消耗降低 90%（仅 10% 查询使用 LLM）
- 成本降低 90%

## 混合检索流程

### 流程说明

混合检索结合三种检索策略，提高字段匹配的准确率和召回率：

**精确匹配（Exact Match）**：
- 策略：字符串完全匹配或同义词匹配
- 权重：1.0（最高优先级）
- 用途：处理明确的字段名称

**向量检索（Vector Search）**：
- 策略：语义相似度（Embedding）
- 权重：0.8
- 用途：处理语义相近的表达

**关键词检索（Keyword Search）**：
- 策略：BM25 算法
- 权重：0.6
- 用途：处理部分匹配和模糊查询

### 执行流程

```
查询: "销售额"
    ↓
┌──────────────────────────────────────────────────────────┐
│ 统一检索器（UnifiedRetriever）                            │
│  strategy = HYBRID                                        │
│  top_k = 20                                               │
└──────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────── 并行检索 ───────────────────────┐
│                                                          │
│  ┌─────────────────────┐  ┌─────────────────────┐      │
│  │ 精确匹配             │  │ 向量检索             │      │
│  │  - "销售额"          │  │  - embedding("销售额")│     │
│  │  - "sales_amount"    │  │  - 余弦相似度        │      │
│  │  ✅ 命中！           │  │  - Top-10 结果       │      │
│  │                      │  │                      │      │
│  │ 结果:                │  │ 结果:                │      │
│  │  Field(id=1,         │  │  Field(id=1,         │      │
│  │    score=1.0)        │  │    score=0.95)       │      │
│  └─────────────────────┘  │  Field(id=5,         │      │
│                            │    score=0.82)       │      │
│  ┌─────────────────────┐  │  Field(id=3,         │      │
│  │ 关键词检索           │  │    score=0.75)       │      │
│  │  - BM25("销售额")    │  └─────────────────────┘      │
│  │  - TF-IDF 权重       │                                │
│  │  - Top-10 结果       │                                │
│  │                      │                                │
│  │ 结果:                │                                │
│  │  Field(id=1,         │                                │
│  │    score=0.90)       │                                │
│  │  Field(id=3,         │                                │
│  │    score=0.85)       │                                │
│  │  Field(id=7,         │                                │
│  │    score=0.70)       │                                │
│  └─────────────────────┘                                │
│                                                          │
└──────────────────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────────────────┐
│ 分数融合（Score Fusion）                                  │
│  策略: 加权平均 + 去重                                    │
│                                                          │
│  Field(id=1):                                            │
│    exact=1.0 * 1.0 = 1.0                                 │
│    vector=0.95 * 0.8 = 0.76                              │
│    keyword=0.90 * 0.6 = 0.54                             │
│    final = (1.0 + 0.76 + 0.54) / 3 = 0.77 ✅ 最高分      │
│                                                          │
│  Field(id=5):                                            │
│    exact=0 * 1.0 = 0                                     │
│    vector=0.82 * 0.8 = 0.66                              │
│    keyword=0 * 0.6 = 0                                   │
│    final = (0 + 0.66 + 0) / 3 = 0.22                     │
│                                                          │
│  Field(id=3):                                            │
│    exact=0 * 1.0 = 0                                     │
│    vector=0.75 * 0.8 = 0.60                              │
│    keyword=0.85 * 0.6 = 0.51                             │
│    final = (0 + 0.60 + 0.51) / 3 = 0.37                  │
└──────────────────────────────────────────────────────────┘
    ↓
┌──────────────────────────────────────────────────────────┐
│ 排序和返回 Top-20                                         │
│                                                          │
│  1. Field(id=1, name="sales_amount", score=0.77)         │
│  2. Field(id=3, name="revenue", score=0.37)              │
│  3. Field(id=5, name="total_sales", score=0.22)          │
│  ...                                                     │
└──────────────────────────────────────────────────────────┘
    ↓
返回给 FieldMapper Agent
```

### 优势

| 检索策略 | 优势 | 劣势 |
|---------|------|------|
| 精确匹配 | 100% 准确（命中时） | 召回率低（仅完全匹配） |
| 向量检索 | 语义理解强 | 可能返回不相关结果 |
| 关键词检索 | 部分匹配能力强 | 无语义理解 |
| **混合检索** | **准确率高 + 召回率高** | 计算开销稍大 |

**性能指标**：
- 准确率：92%（Top-1）
- 召回率：98%（Top-20）
- 延迟：< 50ms（并行检索）

## LangGraph 工作流状态转换

```
START
  ↓
semantic_parser
  ↓
[条件判断]
  ├─ intent=DATA_QUERY → insight
  ├─ intent=CLARIFICATION → END
  └─ intent=IRRELEVANT → END
  ↓
insight
  ↓
replanner
  ↓
[条件判断]
  ├─ decision=CONTINUE → semantic_parser (并行问题)
  └─ decision=FINISH → END
```

## 缓存策略

```
查询请求
  ↓
┌─────────────────────────────────────┐
│ 1. 生成缓存键                        │
│  key = hash(query + data_model_id)  │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 2. 检查缓存                          │
│  Redis.get(key)                      │
└─────────────────────────────────────┘
  ↓
[缓存命中?]
  ├─ YES → 返回缓存结果（延迟 <10ms）
  └─ NO  → 执行查询
      ↓
    ┌─────────────────────────────────────┐
    │ 3. 执行查询                          │
    │  - SemanticParser                    │
    │  - Insight                           │
    │  - Replanner                         │
    └─────────────────────────────────────┘
      ↓
    ┌─────────────────────────────────────┐
    │ 4. 缓存结果                          │
    │  Redis.set(key, result, ttl=3600)    │
    └─────────────────────────────────────┘
      ↓
    返回结果
```

## 错误处理流程

```
查询执行
  ↓
[执行成功?]
  ├─ YES → 返回结果
  └─ NO  → 错误处理
      ↓
    ┌─────────────────────────────────────┐
    │ ReAct 错误处理器                     │
    │  - 分析错误类型                      │
    │  - 决定处理策略                      │
    └─────────────────────────────────────┘
      ↓
    [错误类型]
      ├─ 字段不存在 → CLARIFY（请求用户澄清）
      ├─ 查询超时 → RETRY（重试）
      ├─ 权限错误 → ABORT（终止）
      └─ 其他错误 → RETRY（最多 3 次）
```
