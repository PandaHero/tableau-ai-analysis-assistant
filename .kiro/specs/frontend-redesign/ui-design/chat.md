# 对话界面设计

## 页面结构

系统包含**两个独立页面**，通过页面切换实现对话流程：

```
┌─────────────┐                      ┌─────────────┐
│             │    发送第一条消息     │             │
│    首页     │  ─────────────────►  │  对话页面   │
│  (欢迎页)   │                      │             │
│             │  ◄─────────────────  │             │
└─────────────┘    点击返回按钮      └─────────────┘
```

## 页面切换逻辑

| 当前页面 | 动作 | 结果 |
|---------|------|------|
| 首页 | 发送问题 | 跳转到对话页面，开始对话 |
| 首页 | 点击示例问题 | 填入并发送，跳转到对话页面 |
| 对话页面 | 点击返回 | 回到首页，清空当前对话（新 session） |
| 对话页面 | 继续发送问题 | 保持在对话页面，追加消息 |

## 组件设计规范

### 分析卡片

| 元素 | 样式 | 说明 |
|-----|------|------|
| 卡片容器 | 白色背景，1px #E8E8E8 边框，8px 圆角 | 每轮分析一个卡片 |
| ❓ 问题标题 | 16px 加粗，#1A1A1A，卡片头部 | 多轮时显示 |
| 分隔线 | 虚线 `─ ─ ─`，#E0E0E0 | 区分表格和发现 |

### 数据表格工具栏

```
[🔧查询详情]            共 25 条  ◀  1/3  ▶            [📥导出]
 ↑                        ↑          ↑                    ↑
次要功能                 总数      分页控件            主要操作
(左对齐)               (居中)     (居中)              (右对齐)
```

| 元素 | 位置 | 样式 |
|-----|------|------|
| [🔧查询详情] | 左侧 | 文字按钮，#666666，hover 变 #1F77B4 |
| 共 N 条 | 中间偏左 | 12px，#999999 |
| ◀ 1/3 ▶ | 中间 | 分页控件，当前页加粗 |
| [📥导出] | 右侧 | 主按钮样式，#1F77B4 背景 |

**分页规则：**
- 数据 ≤ 10 条：不显示分页控件，只显示 `共 N 条`
- 数据 > 10 条：显示完整工具栏

## 展示顺序（按节点输出顺序）

```
1. 📊 查询结果（Execute 节点输出，最先到达）
   - 数据表格（每页 10 条）
   - 工具栏：[🔧查询详情] | 共N条 ◀ 1/3 ▶ | [📥导出]
   ↓
2. 💡 发现（Insight Agent 输出）
   - 列表形式，异常用 ⚠️ 标记
   ↓
3. 💭 思考气泡（Replanner 输出，should_replan=true 时）
   - 解释为什么要进行下一轮分析
   - 连接线 │ ▼ 连接到下一轮
   ↓
4. [重复 1-3，直到 should_replan=false]
   ↓
5. ════ 双线分隔 ════
   ↓
6. 📝 总结（Replanner 输出，should_replan=false 时）
   - Markdown 格式，汇总所有轮次的发现
   ↓
7. ──── 单线分隔 ────
   ↓
8. 💬 继续探索（Replanner 输出）
   - 卡片式推荐问题，最多显示 3 个
```

## 复制功能

- 位置：AI 消息卡片头部右侧
- 默认：显示 [📋复制]
- 点击后：变为 [✓ 已复制]，2 秒后恢复
- 复制内容：纯文本格式（去除 Markdown 标记）
