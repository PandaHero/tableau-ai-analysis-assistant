## 设计理念

### 核心原则

1. **结果导向** - 用户只关心分析结果，不关心执行过程
2. **对话式交互** - 像和分析师对话一样自然
3. **渐进式披露** - 先给结论，细节按需展开
4. **Tableau 原生风格** - 融入 Tableau 的设计语言

### 设计参考

| 产品 | 借鉴点 |
|------|--------|
| **ChatGPT Data Analysis** | 自然对话、代码/结果分离、可折叠的技术细节 |
| **Claude** | 极简消息设计、Artifacts 侧边展示、思考过程简洁 |
| **Perplexity** | 来源引用、答案优先、细节折叠 |
| **Julius AI** | 数据分析专用、图表内嵌、洞察突出 |

### 关键设计决策

❌ **不展示**：Agent 执行流程、节点状态、技术细节
✅ **展示**：思考摘要、分析结论、数据结果、业务洞察

## AI 消息设计

### 消息结构

AI 回复采用**结构化 Markdown** 格式，包含以下部分：

```markdown
[开场白 - 简短说明要做什么]

## 📊 分析结果
[核心数据/图表]

## 💡 关键发现
[洞察列表，按重要性排序]

[可折叠的技术细节]

---
💬 您还想了解什么？
[推荐的后续问题]
```

### 设计原则

1. **开场白简洁** - 一句话说明分析意图，不解释技术过程
2. **结果优先** - 数据和图表放在最前面
3. **洞察突出** - 用编号列表，加粗关键信息
4. **技术细节隐藏** - VizQL 查询等默认折叠
5. **引导继续** - 提供推荐问题，降低用户思考成本

## 思考过程展示

### 方案：简洁的状态提示

**不展示**：Agent 流程图、节点状态、执行时间
**展示**：简洁的思考状态文字

**思考中状态**：
```
┌──────────────────────────────────────────────────┐
│ 🤖                                                │
│                                                   │
│ 正在分析数据...                                   │
│ ● ● ●                                            │
└──────────────────────────────────────────────────┘
```

**分步骤提示（可选，流式输出时）**：
```
┌──────────────────────────────────────────────────┐
│ 🤖                                                │
│                                                   │
│ ✓ 理解您的问题                                   │
│ ✓ 构建查询                                       │
│ ● 执行分析中...                                  │
└──────────────────────────────────────────────────┘
```

### 实现方式

```typescript
// 简洁的状态消息
const statusMessages = {
  understanding: '理解您的问题...',
  building: '构建查询...',
  executing: '执行分析中...',
  analyzing: '生成洞察...',
};

// 流式输出时，只显示当前步骤
const currentStatus = ref('正在分析数据...');
```

## 推荐问题设计

### 设计原则

每次分析完成后，提供 2-3 个推荐的后续问题，帮助用户深入分析。

**来源**：
- Replanner Agent 生成的 `next_questions`
- 基于当前分析结果的智能推荐

**展示方式**：
```
---
💬 您还想了解什么？

• 哪个省份增长最快？
• 对比去年同期表现如何？
• 分析 Q4 增长的原因
```

**交互**：
- 点击推荐问题，自动填入输入框并发送
- 用户也可以忽略，输入自己的问题

## 输入区域设计

### 简洁设计

```
┌────────────────────────────────────────────────────────┐
│ [输入您的分析问题...]                         [发送]   │

**特点**：

```

## 流式输出实现

### SSE 事件处理

```typescript
// 简化的状态管理
const state = ref({
  isThinking: false,
  thinkingText: '正在分析...',
  content: '',
});

// 监听事件
client.on(EventType.WORKFLOW_START, () => {
  state.value.isThinking = true;
  state.value.thinkingText = '正在分析...';
});

client.on(EventType.TOKEN, (event) => {
  state.value.content += event.data.token;
});

client.on(EventType.WORKFLOW_COMPLETE, () => {
  state.value.isThinking = false;
});
```

### 思考状态组件

```vue
<template>
  <div v-if="isThinking" class="thinking-indicator">
    <span class="text">{{ thinkingText }}</span>
    <span class="dots">
      <span></span>
      <span></span>
      <span></span>
    </span>
  </div>
</template>
```

## 与原设计的对比

| 方面 | 原设计 | 新设计 |
|------|--------|--------|
| **执行流程** | 展示 Agent 流程图 | ❌ 不展示 |
| **节点状态** | 展示每个节点状态 | ❌ 不展示 |
| **执行时间** | 展示每个节点耗时 | 折叠在技术细节中 |
| **思考过程** | 详细的步骤展示 | 简洁的状态文字 |
| **分析结果** | 需要展开查看 | ✅ 直接展示 |
| **洞察** | 需要展开查看 | ✅ 直接展示 |
| **技术细节** | 默认展示 | 默认折叠 |

## 总结

这个设计方案的核心理念是：

1. **用户不关心怎么做，只关心结果** - 隐藏执行流程
2. **像和分析师对话** - 自然的对话式交互
3. **结果优先** - 数据和洞察直接展示
4. **技术细节按需** - VizQL 等默认折叠
5. **引导深入** - 推荐问题降低用户思考成本

参考了 ChatGPT Data Analysis、Claude、Perplexity、Julius AI 等主流产品的设计思路。
