**典型需求**：
- 快速查询业务指标
- 对比不同维度的数据
- 发现异常和趋势
- 生成分析报告

**使用场景**：
- 每日业务回顾
- 周报/月报生成
- 临时数据查询
- 根因分析

## 典型场景

### 场景1：日常业务回顾

**用户角色**：业务数据分析师

**业务背景**：
- 每天早上需要查看昨日的业务数据
- 关注销售额、订单量、客户数等关键指标
- 需要对比不同地区和产品类别的表现

**用户问题**：
```
昨日各地区的销售额、订单量和客户数
```

**系统执行流程**：

1. **问题理解**：
   - 问题类型：对比
   - 时间范围：昨日（相对时间）
   - 维度：地区
   - 度量：销售额、订单量、客户数
   - 复杂度：Simple

2. **字段选择**：
   - 维度：地区（level=1，粗粒度）
   - 度量：销售额（SUM）、订单量（COUNT）、客户数（COUNTD）

3. **任务拆分**：
   - 不需要拆分（一个查询即可）

4. **查询执行**：
   ```json
   {
     "fields": [
       {"fieldCaption": "地区"},
       {"fieldCaption": "销售额", "function": "SUM"},
       {"fieldCaption": "订单量", "function": "COUNT"},
       {"fieldCaption": "客户数", "function": "COUNTD"}
     ],
     "filters": [
       {
         "fieldCaption": "订单日期",
         "filterType": "categorical",
         "values": ["2025-01-14"]
       }
     ]
   }
   ```

5. **洞察生成**：
   - 关键发现：华东地区销售额最高，达到50万元
   - 异常：西北地区订单量为0（可能是数据延迟）
   - 建议：关注西北地区的数据质量

6. **最终报告**：
   ```
   昨日各地区业务表现：

   1. 华东地区表现最好，销售额50万元，订单量1200单，客户数800人
   2. 华北地区次之，销售额40万元，订单量1000单，客户数700人
   3. 西北地区异常，订单量为0，建议检查数据质量

   建议：
   - 关注西北地区的数据质量问题
   - 分析华东地区的成功经验，推广到其他地区
   ```

### 场景2：同比分析

**用户角色**：业务经理

**业务背景**：
- 每月初需要对比本月和去年同期的业务数据
- 关注销售额增长率和市场份额变化
- 需要识别增长最快和最慢的地区

**用户问题**：
```
2024年1月vs 2023年1月的销售额增长率，按地区分析
```

**系统执行流程**：

1. **问题理解**：
   - 问题类型：对比、同比
   - 时间范围：2024年1月 vs 2023年1月
   - 维度：地区
   - 度量：销售额、增长率
   - 复杂度：Medium
   - 隐含需求：需要两个时间段的数据

2. **字段选择**：
   - 维度：地区（level=1，粗粒度）
   - 度量：销售额（SUM）

3. **任务拆分**：
   - 子任务1：2024年1月各地区的销售额
   - 子任务2：2023年1月各地区的销售额
   - 合并：计算增长率

4. **查询执行**（并行）：

   **查询1**：
   ```json
   {
     "fields": [
       {"fieldCaption": "地区"},
       {"fieldCaption": "销售额", "function": "SUM"}
     ],
     "filters": [
       {
         "fieldCaption": "订单日期",
         "function": "YEAR",
         "values": ["2024"]
       },
       {
         "fieldCaption": "订单日期",
         "function": "MONTH",
         "values": ["1"]
       }
     ]
   }
   ```

   **查询2**：
   ```json
   {
     "fields": [
       {"fieldCaption": "地区"},
       {"fieldCaption": "销售额", "function": "SUM"}
     ],
     "filters": [
       {
         "fieldCaption": "订单日期",
         "function": "YEAR",
         "values": ["2023"]
       },
       {
         "fieldCaption": "订单日期",
         "function": "MONTH",
         "values": ["1"]
       }
     ]
   }
   ```

5. **数据合并**：
   ```python
   # 合并两个查询结果
   merged_data = pd.merge(
       query1_result,
       query2_result,
       on="地区",
       suffixes=("_2024", "_2023")
   )

   # 计算增长率
   merged_data["增长率"] = (
       (merged_data["销售额_2024"] - merged_data["销售额_2023"])
       / merged_data["销售额_2023"]
   )
   ```

6. **洞察生成**：
   - 关键发现：华东地区增长率最高，达到25%
   - 异常：西北地区增长率为负，下降了10%
   - 建议：分析华东地区的增长驱动因素，诊断西北地区的下降原因

7. **重规划决策**：
   - 发现西北地区增长率异常低
   - 生成新问题："西北地区各产品类别的销售额增长率"

8. **第2轮分析**（重规划）：
   - 问题：西北地区各产品类别的销售额增长率
   - 发现：家具类别下降了30%，办公用品增长了10%
   - 建议：重点关注家具类别的销售策略

9. **最终报告**：
   ```
   2024年1月vs 2023年1月销售额同比分析：

   总体表现：
   - 全国销售额增长15%，从1000万元增长到1150万元

   地区表现：
   1. 华东地区增长最快，增长率25%（从400万元增长到500万元）
   2. 华北地区增长15%（从300万元增长到345万元）
   3. 西北地区下降10%（从100万元下降到90万元）

   深入分析（西北地区）：
   - 家具类别下降30%，是主要拖累因素
   - 办公用品增长10%，表现良好

   建议：
   1. 分析华东地区的增长驱动因素，推广到其他地区
   2. 重点关注西北地区家具类别的销售策略
   3. 考虑调整西北地区的产品组合
   ```

### 场景3：异常诊断

**用户角色**：业务数据分析师

**业务背景**：
- 发现某个地区的利润率异常低
- 需要诊断原因
- 需要多层级下钻分析

**用户问题**：
```
为什么华东地区的利润率只有8%，低于平均水平12%？
```

**系统执行流程**：

1. **问题理解**：
   - 问题类型：诊断
   - 维度：地区（华东）
   - 度量：利润率
   - 复杂度：Complex
   - 隐含需求：需要多层级下钻

2. **第1轮分析**：各地区的利润率
   - 发现：华东地区利润率8%，全国平均12%
   - 建议：下钻到产品类别

3. **第2轮分析**（重规划）：华东地区各产品类别的利润率
   - 发现：电子产品利润率只有5%，家具利润率15%
   - 建议：下钻到电子产品的子类别

4. **第3轮分析**（重规划）：华东地区电子产品各子类别的利润率
   - 发现：手机利润率只有2%，电脑利润率8%
   - 建议：分析手机的定价策略

5. **最终报告**：
   ```
   华东地区利润率低的根因分析：

   分析路径：
   1. 各地区利润率对比 → 发现华东地区偏低
   2. 华东地区各产品类别利润率 → 发现电子产品偏低
   3. 华东地区电子产品各子类别利润率 → 发现手机偏低

   根因：
   - 华东地区手机类别的利润率只有2%，远低于其他地区的8%
   - 手机类别占华东地区销售额的40%，是主要拖累因素

   可能原因：
   1. 华东地区手机市场竞争激烈，价格战严重
   2. 华东地区手机的成本结构不合理
   3. 华东地区手机的产品组合偏向低端产品

   建议：
   1. 优化华东地区手机的定价策略，避免过度价格战
   2. 分析华东地区手机的成本结构，寻找降本空间
   3. 调整华东地区手机的产品组合，增加高端产品占比
   ```

### 场景4：探索式分析

**用户角色**：数据科学家

**业务背景**：
- 需要探索数据中的模式和趋势
- 不确定具体要分析什么
- 需要系统智能推荐分析方向

**用户问题**：
```
2024年的业务数据有什么值得关注的趋势和异常？
```

**系统执行流程**：

1. **问题理解**：
   - 问题类型：探索、趋势、异常
   - 时间范围：2024年
   - 复杂度：Complex
   - 隐含需求：需要多维度探索

2. **第1轮分析**：2024年各月的销售额趋势
   - 发现：销售额整体上升，但12月异常下降
   - 建议：分析12月的异常原因

3. **第2轮分析**（重规划）：2024年12月各地区的销售额
   - 发现：华东地区12月销售额下降30%
   - 建议：分析华东地区12月的异常原因

4. **第3轮分析**（重规划）：华东地区12月各产品类别的销售额
   - 发现：电子产品12月销售额下降50%
   - 建议：分析电子产品12月的异常原因

5. **第4轮分析**（重规划）：华东地区12月电子产品的订单量和客户数
   - 发现：订单量下降40%，客户数下降30%
   - 可能原因：促销活动结束、竞争对手促销、季节性因素

6. **最终报告**：
   ```
   2024年业务数据探索分析：

   关键发现：
   1. 销售额整体上升，全年增长15%
   2. 12月销售额异常下降，环比下降20%

   异常诊断：
   - 12月销售额下降主要由华东地区驱动
   - 华东地区12月销售额下降30%
   - 华东地区电子产品12月销售额下降50%
   - 订单量和客户数同步下降

   可能原因：
   1. 11月促销活动透支了12月的需求
   2. 竞争对手在12月推出了更有吸引力的促销
   3. 电子产品的季节性因素（12月不是购买旺季）

   建议：
   1. 分析11月促销活动的影响，优化促销节奏
   2. 监控竞争对手的促销策略，及时调整
   3. 考虑在12月推出针对性的促销活动
   4. 分析其他地区12月的表现，学习成功经验
   ```

## 最佳实践

### 1. 问题表达

**清晰的问题**：
- ✅ "2024年各地区的销售额"
- ✅ "最近3个月的销售额趋势"
- ✅ "销售额Top 10的门店"

**模糊的问题**：
- ❌ "看一下数据"
- ❌ "分析一下"
- ❌ "有什么问题"

**建议**：
- 明确时间范围
- 明确分析维度
- 明确分析指标
- 明确分析目的

---

### 2. 渐进式分析

**从总体到明细**：
1. 先看总体趋势
2. 再看异常和关键发现
3. 最后下钻到明细

**示例**：
```
第1轮：各地区的销售额 → 发现华东地区最高
第2轮：华东地区各产品类别的销售额 → 发现电子产品最高
第3轮：华东地区电子产品各子类别的销售额 → 发现手机最高
```

---

### 3. 利用重规划

**主动探索**：
- 系统会根据分析结果推荐后续问题
- 用户可以选择接受或修改推荐问题
- 支持多轮重规划，深入分析

**示例**：
```
用户问题：2024年各地区的销售额
系统推荐：华东地区销售额最高，是否需要分析华东地区各产品类别的销售额？
用户选择：是
系统执行：华东地区各产品类别的销售额
```

---

### 4. 关注数据质量

**数据质量指标**：
- 完整性：缺失值占比
- 一致性：重复值占比
- 准确性：异常值占比
- 时效性：数据更新时间

**建议**：
- 关注数据质量评分
- 对低质量数据保持警惕
- 及时反馈数据质量问题
