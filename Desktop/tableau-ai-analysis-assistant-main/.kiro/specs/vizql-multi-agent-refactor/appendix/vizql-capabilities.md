# VizQL查询能力详解

本文档详细说明VizQL Data Service的查询能力边界、问题拆分决策指南和最佳实践。

---

## VizQL核心能力

### 1. 多维度查询

**能力说明**：一个VizQL查询可以包含多个维度字段

**示例**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "产品类别"},
    {"fieldCaption": "门店"}
  ]
}
```

**业务场景**：
- "各地区各产品类别的销售额" ✅ 一个查询即可
- "各地区各门店的销售额" ✅ 一个查询即可

**注意事项**：
- 维度越多，结果行数越多（笛卡尔积）
- 建议维度数量 <= 3个，避免结果过大

---

### 2. 多度量查询

**能力说明**：一个VizQL查询可以包含多个度量字段

**示例**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"},
    {"fieldCaption": "利润", "function": "SUM"},
    {"fieldCaption": "订单量", "function": "COUNT"}
  ]
}
```

**业务场景**：
- "各地区的销售额、利润和订单量" ✅ 一个查询即可
- "各地区的销售额和利润率" ✅ 一个查询即可（利润率可以用计算字段）

**注意事项**：
- 所有度量必须使用相同的聚合粒度（相同的维度组合）
- 如果需要不同粒度的度量，需要拆分为多个查询

---

### 3. 多筛选条件

**能力说明**：一个VizQL查询可以包含多个筛选条件

**示例**：
```json
{
  "filters": [
    {
      "fieldCaption": "订单日期",
      "function": "YEAR",
      "filterType": "categorical",
      "values": ["2016"]
    },
    {
      "fieldCaption": "地区",
      "filterType": "categorical",
      "values": ["北京", "上海"]
    },
    {
      "fieldCaption": "销售额",
      "filterType": "quantitative",
      "min": 1000
    }
  ]
}
```

**业务场景**：
- "2016年北京和上海的销售额>1000的订单" ✅ 一个查询即可
- "2016年各地区的销售额，筛选销售额>1000" ✅ 一个查询即可

**注意事项**：
- 筛选条件之间是AND关系
- 如果需要OR关系，需要拆分为多个查询

---

### 4. 聚合函数

**支持的聚合函数**：
- **SUM** - 求和
- **AVG** - 平均值
- **COUNT** - 计数
- **MIN** - 最小值
- **MAX** - 最大值
- **COUNTD** - 去重计数

**示例**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"},
    {"fieldCaption": "订单量", "function": "COUNT"},
    {"fieldCaption": "客户数", "function": "COUNTD"}
  ]
}
```

**业务场景**：
- "各地区的总销售额" → SUM(销售额)
- "各地区的平均销售额" → AVG(销售额)
- "各地区的订单数" → COUNT(订单ID)
- "各地区的客户数" → COUNTD(客户ID)

---

### 5. 日期函数

**支持的日期函数**：
- **YEAR** - 年份
- **MONTH** - 月份
- **DAY** - 日期
- **QUARTER** - 季度
- **WEEK** - 周
- **TRUNC_YEAR** - 截断到年初
- **TRUNC_MONTH** - 截断到月初
- **TRUNC_QUARTER** - 截断到季初

**示例**：
```json
{
  "fields": [
    {"fieldCaption": "订单日期", "function": "YEAR"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ]
}
```

**业务场景**：
- "各年的销售额" → YEAR(订单日期)
- "各月的销售额" → MONTH(订单日期)
- "各季度的销售额" → QUARTER(订单日期)

**注意事项**：
- 日期函数只能应用于日期类型的字段
- 日期函数会改变聚合粒度

---

### 6. 计算字段

**能力说明**：支持自定义计算表达式

**示例**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"},
    {"fieldCaption": "利润", "function": "SUM"},
    {
      "fieldCaption": "利润率",
      "calculation": "SUM([利润]) / SUM([销售额])"
    }
  ]
}
```

**业务场景**：
- "各地区的利润率" → 利润 / 销售额
- "各地区的增长率" → (当期 - 上期) / 上期

**注意事项**：
- 计算字段可以引用其他字段
- 计算字段不能与function同时使用

---

### 7. 排序

**能力说明**：支持多字段排序，指定优先级

**示例**：
```json
{
  "orderBy": [
    {
      "fieldCaption": "销售额",
      "direction": "DESC",
      "sortPriority": 1
    },
    {
      "fieldCaption": "地区",
      "direction": "ASC",
      "sortPriority": 2
    }
  ]
}
```

**业务场景**：
- "按销售额降序排列" → sortPriority=1, direction=DESC
- "先按销售额降序，再按地区升序" → 两个排序规则

**注意事项**：
- sortPriority必须唯一且连续（1, 2, 3...）
- direction只能是ASC或DESC

---

### 8. TopN筛选

**能力说明**：支持TopNFilter限制结果数量

**示例**：
```json
{
  "topNFilter": {
    "field": "销售额",
    "top": 10,
    "direction": "DESC"
  }
}
```

**业务场景**：
- "销售额Top 10的门店" ✅
- "利润率最低的5个地区" ✅

**注意事项**：
- TopN筛选在聚合后应用
- TopN筛选会影响结果的可比性（不同TopN不能直接对比）

## 问题拆分决策指南

### 不需要拆分的情况（一个VizQL查询即可）

#### 1. 多个度量 + 相同维度组合 + 相同时间段

**示例问题**：
- "2016年各地区的销售额、利润和订单量"
- "各产品类别的销售额和利润率"

**VizQL查询**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"},
    {"fieldCaption": "利润", "function": "SUM"},
    {"fieldCaption": "订单量", "function": "COUNT"}
  ],
  "filters": [
    {"fieldCaption": "订单日期", "function": "YEAR", "values": ["2016"]}
  ]
}
```

**拆分决策**：✅ 不需要拆分

---

#### 2. 多个维度 + 多个度量

**示例问题**：
- "各地区各产品类别的销售额和利润"
- "各门店各月的销售额"

**VizQL查询**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "产品类别"},
    {"fieldCaption": "销售额", "function": "SUM"},
    {"fieldCaption": "利润", "function": "SUM"}
  ]
}
```

**拆分决策**：✅ 不需要拆分

---

#### 3. 多个筛选条件 + 单个时间段

**示例问题**：
- "2016年北京和上海的销售额>1000的订单"
- "各地区的销售额，筛选产品类别=家具且利润率>10%"

**VizQL查询**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ],
  "filters": [
    {"fieldCaption": "订单日期", "function": "YEAR", "values": ["2016"]},
    {"fieldCaption": "地区", "values": ["北京", "上海"]},
    {"fieldCaption": "销售额", "min": 1000}
  ]
}
```

**拆分决策**：✅ 不需要拆分

---

### 需要拆分的情况（需要多个VizQL查询）

#### 1. 多个时间段对比（同比、环比）

**示例问题**：
- "2016年vs 2015年的销售额对比"
- "本月vs上月的销售额增长率"

**拆分理由**：
- VizQL不支持在一个查询中同时查询多个不连续的时间段
- 需要分别查询两个时间段，然后合并结果

**拆分方案**：
```
子任务1：2016年的销售额
子任务2：2015年的销售额
合并：计算增长率
```

**VizQL查询1**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ],
  "filters": [
    {"fieldCaption": "订单日期", "function": "YEAR", "values": ["2016"]}
  ]
}
```

**VizQL查询2**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ],
  "filters": [
    {"fieldCaption": "订单日期", "function": "YEAR", "values": ["2015"]}
  ]
}
```

**拆分决策**：❌ 需要拆分

---

#### 2. 不同维度组合（先总体后明细）

**示例问题**：
- "先看各地区的销售额，再看华东地区各门店的销售额"
- "先看各产品类别的销售额，再看家具类别各产品的销售额"

**拆分理由**：
- 总体和明细的维度组合不同
- 需要先查询总体，再根据结果决定是否查询明细

**拆分方案**：
```
子任务1：各地区的销售额（粗粒度）
子任务2：华东地区各门店的销售额（细粒度，依赖子任务1的结果）
```

**VizQL查询1**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ]
}
```

**VizQL查询2**：
```json
{
  "fields": [
    {"fieldCaption": "门店"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ],
  "filters": [
    {"fieldCaption": "地区", "values": ["华东"]}
  ]
}
```

**拆分决策**：❌ 需要拆分

---

#### 3. 计算依赖（需要先计算A再计算B）

**示例问题**：
- "各地区的销售额占比"（需要先计算总销售额）
- "各地区的销售额排名"（需要先计算所有地区的销售额）

**拆分理由**：
- 占比需要先计算总计
- 排名需要先计算所有值

**拆分方案**：
```
子任务1：各地区的销售额
子任务2：总销售额
合并：计算占比
```

**VizQL查询1**：
```json
{
  "fields": [
    {"fieldCaption": "地区"},
    {"fieldCaption": "销售额", "function": "SUM"}
  ]
}
```

**VizQL查询2**：
```json
{
  "fields": [
    {"fieldCaption": "销售额", "function": "SUM"}
  ]
}
```

**拆分决策**：❌ 需要拆分

---

#### 4. 不同筛选条件组合对比

**示例问题**：
- "家具类别vs办公用品类别的销售额对比"
- "北京vs上海的销售额对比"

**拆分理由**：
- 虽然可以在一个查询中用维度分组实现
- 但如果需要对比的是不同筛选条件的结果，建议拆分

**拆分方案**：
```
子任务1：家具类别的销售额
子任务2：办公用品类别的销售额
合并：对比结果
```

**拆分决策**：⚠️ 可选拆分（取决于具体需求）

## 最佳实践

### 1. 优先使用单个查询

**原则**：能用一个查询解决的，不要拆分

**理由**：
- 减少API调用次数
- 减少数据合并的复杂度
- 提高查询效率

**示例**：
- ✅ "各地区的销售额、利润和订单量" → 一个查询
- ❌ 不要拆分为3个查询（销售额、利润、订单量）

---

### 2. 合理使用TopN筛选

**原则**：TopN筛选在聚合后应用，注意可比性

**理由**：
- TopN筛选会影响结果的可比性
- 不同TopN的结果不能直接对比

**示例**：
- ✅ "销售额Top 10的门店" → 使用TopN筛选
- ❌ "各地区销售额Top 10的门店" → 不要使用TopN筛选（会导致不同地区的Top 10不可比）

---

### 3. 避免过多维度

**原则**：维度数量 <= 3个

**理由**：
- 维度越多，结果行数越多（笛卡尔积）
- 结果过大会影响性能和用户体验

**示例**：
- ✅ "各地区各产品类别的销售额" → 2个维度
- ⚠️ "各地区各产品类别各门店的销售额" → 3个维度（可能结果过大）
- ❌ "各地区各产品类别各门店各客户的销售额" → 4个维度（不推荐）

---

### 4. 合理使用计算字段

**原则**：简单计算用计算字段，复杂计算用代码

**理由**：
- 计算字段在VizQL中执行，性能较好
- 复杂计算在代码中执行，更灵活

**示例**：
- ✅ "利润率" → 使用计算字段：`SUM([利润]) / SUM([销售额])`
- ❌ "增长率" → 不要使用计算字段（需要两个时间段的数据，应该在代码中计算）

## 常见问题

### Q1: 为什么同比分析需要拆分？

**A**: VizQL不支持在一个查询中同时查询多个不连续的时间段。同比分析需要查询当期和去年同期两个时间段，因此需要拆分为两个查询。

---

### Q2: 为什么占比分析需要拆分？

**A**: 占比分析需要先计算总计，然后计算每个部分占总计的百分比。VizQL不支持在一个查询中同时计算总计和分组聚合，因此需要拆分为两个查询。

---

### Q3: 如何判断是否需要拆分？

**A**: 遵循以下原则：
1. 如果需要多个不连续的时间段 → 拆分
2. 如果需要不同粒度的维度组合 → 拆分
3. 如果需要先计算A再计算B → 拆分
4. 其他情况 → 不拆分

---

### Q4: TopN筛选什么时候使用？

**A**:
- ✅ 单一维度的TopN（如"销售额Top 10的门店"）
- ❌ 多维度的TopN（如"各地区销售额Top 10的门店"）→ 会导致不同地区的Top 10不可比

---

### Q5: 如何处理相对时间？

**A**:
- AI只负责理解和提取相对时间表达（如"最近3个月"）
- 具体日期计算由查询构建器完成（使用当前日期作为基准）
- 这样可以确保查询的可重复性和准确性
