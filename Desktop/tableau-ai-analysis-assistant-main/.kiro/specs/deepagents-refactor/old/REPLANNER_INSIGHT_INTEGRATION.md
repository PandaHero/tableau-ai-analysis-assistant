# 重规划与洞察的正确集成方案

## 🎯 核心问题重新思考

你提出的问题非常关键：
1. **洞察的数据来源是什么？**
2. **重规划到底是干什么的？**
3. **评估是否重规划这个怎么跟洞察结合？**

让我基于现有设计文档深入分析。

---

## 📚 关键理解：三个不同的概念

### 1. 累积洞察（Progressive Insight Analysis）

**本质**：针对**单个查询结果**的智能分析策略

**场景**：
```
用户问："2016年各地区的销售额"

执行查询 → 返回 10,000 行数据

累积洞察分析：
├─ 第1块：分析 Top 100 行
│   └─ 发现：华东地区第一，销售额 1000万
├─ 第2块：分析 101-500 行
│   └─ 发现：大部分地区在 50-100万之间
├─ AI 判断：已经得到足够洞察
└─ 早停：不再分析剩余数据

输出：华东地区销售额最高，显著领先其他地区
```

**关键点**：
- ✅ 针对**已有数据**的分析
- ✅ 不执行新查询
- ✅ 解决大数据的 Token 问题
- ✅ 智能早停

---

### 2. 重规划（Replanning）

**本质**：**重新生成查询计划**，执行新的查询

**场景**：
```
用户问："为什么华东地区利润率低？"

第1轮：
├─ Task Planner 生成查询计划：
│   ├─ q1: 查询华东地区利润率
│   └─ q2: 查询全国平均利润率
├─ 执行查询
├─ Insight Agent 分析：
│   └─ 华东利润率 12%，低于全国平均 15%
│   └─ 结论：确认华东利润率低，但不知道原因 ❌
└─ Replan Agent 判断：
    └─ 需要更多维度的数据 → 重规划

第2轮（重规划）：
├─ Task Planner 重新生成查询计划：
│   ├─ q1: 查询华东地区利润率（复用缓存）
│   ├─ q3: 查询华东各产品类别利润率（新查询）
│   └─ q4: 查询华东各门店利润率（新查询）
├─ 执行新查询
├─ Insight Agent 分析：
│   └─ 家具类利润率仅 8%，是主要原因
│   └─ 结论：找到了原因，但还需要知道为什么家具类低 ❌
└─ Replan Agent 判断：
    └─ 需要更深入的数据 → 再次重规划

第3轮（再次重规划）：
├─ Task Planner 重新生成查询计划：
│   ├─ q5: 查询家具类的成本结构（新查询）
│   └─ q6: 查询家具类的折扣情况（新查询）
├─ 执行新查询
├─ Insight Agent 分析：
│   └─ 家具类折扣率 22%，过高
│   └─ 结论：找到根因了 ✅
└─ Replan Agent 判断：
    └─ 已经找到根因 → 不再重规划

输出：完整的根因分析
```

**关键点**：
- ✅ **执行新查询**
- ✅ 重新生成查询计划
- ✅ 利用缓存避免重复查询
- ✅ 解决"查询计划不够好"的问题

---

### 3. 查询结果缓存

**本质**：避免重规划时重复查询

**场景**：
```
第1轮：
├─ q1: 查询华东地区利润率 → 结果缓存 ✅
└─ q2: 查询全国平均利润率 → 结果缓存 ✅

第2轮（重规划）：
├─ q1: 查询华东地区利润率 → 缓存命中，直接返回 ⚡
├─ q3: 查询华东各产品类别利润率 → 新查询，执行
└─ q4: 查询华东各门店利润率 → 新查询，执行

性能提升：
└─ 不需要重新查询 q1，节省时间和资源
```

**关键点**：
- ✅ 重规划时复用已有数据
- ✅ 不是支持"补充查询"
- ✅ 提升性能

## 🎯 关键问题的答案

### Q1: 洞察的数据来源是什么？

**答案**：查询结果

- **Insight Agent 的输入**：每轮查询的结果
- **第1轮**：分析 q1 和 q2 的结果
- **第2轮**：分析 q3 和 q4 的结果（q1 复用缓存）
- **第3轮**：分析 q5 和 q6 的结果

**Insight Agent 不会"发现数据不足"，它只分析已有数据！**

---

### Q2: 重规划到底是干什么的？

**答案**：重新生成查询计划，执行新查询

**触发条件**：
1. ✅ 问题没有充分回答
2. ✅ 需要更多/不同维度的数据
3. ✅ 需要更细粒度的数据
4. ✅ 发现新的分析方向

**不是触发条件**：
- ❌ 用户不满意（用户不参与重规划决策）
- ❌ 数据质量问题（这是查询验证的职责）
- ❌ 单纯的"数据不足"（需要明确缺什么数据）

**重规划的本质**：
- 分析级的决策
- AI 驱动的迭代分析
- 类似人类分析师的思考过程：
  ```
  "我先看看整体情况" → 第1轮
  "哦，华东利润率低，那我看看是哪个产品" → 第2轮（重规划）
  "原来是家具类，那我看看为什么" → 第3轮（再次重规划）
  "找到了，是折扣太高" → 完成
  ```

---

### Q3: 评估是否重规划怎么跟洞察结合？

**答案**：Replanner Agent 基于累积洞察判断

**Replanner Agent 的输入**：
```python
{
    "original_question": "为什么华东地区利润率低？",
    "accumulated_insights": [
        {
            "round": 1,
            "insights": ["华东利润率 12%，低于全国平均 15%"]
        },
        {
            "round": 2,
            "insights": ["家具类利润率仅 8%，是主要原因"]
        }
    ],
    "current_round": 2,
    "max_rounds": 3
}
```

**Replanner Agent 的判断逻辑**：
```
AI 思考：
1. 原始问题是什么？
   → "为什么华东地区利润率低？"
   
2. 当前已经知道什么？
   → 第1轮：华东利润率确实低（12% vs 15%）
   → 第2轮：家具类是主要原因（8%）
   
3. 问题是否充分回答？
   → ❌ 只知道"家具类低"，不知道"为什么家具类低"
   
4. 需要什么数据才能回答？
   → 需要家具类的成本和折扣数据
   
5. 决策：
   → need_replan = true
   → new_question = "家具类的成本结构和折扣情况是怎样的？"
```

**关键点**：
- ✅ Replanner 看到的是**所有轮次的累积洞察**
- ✅ Replanner 判断**问题是否充分回答**
- ✅ Replanner 生成**新问题**（不是新查询）
- ✅ Planning Agent 根据新问题生成新查询计划

## 🔄 与 DeepAgents 的集成

### 在 DeepAgents 中的实现

```python
# 主 Agent 流程
async def main_agent_flow(question):
    current_round = 1
    max_rounds = 3
    accumulated_insights = []
    
    while current_round <= max_rounds:
        # 1. 理解问题（第1轮）或使用新问题（重规划后）
        if current_round == 1:
            understanding = await task("understanding-agent", question)
            current_question = question
        else:
            # 重规划后使用新问题
            current_question = replan_decision.new_question
            understanding = await task("understanding-agent", current_question)
        
        # 2. 生成查询计划
        query_plan = await task("planning-agent", understanding)
        
        # 3. 执行查询（自动利用缓存）
        query_results = []
        for subtask in query_plan.subtasks:
            # 检查缓存
            cached_result = check_cache(subtask.query_key)
            if cached_result:
                query_results.append(cached_result)
            else:
                result = await vizql_query(subtask.query, datasource_luid)
                save_cache(subtask.query_key, result)
                query_results.append(result)
        
        # 4. 渐进式洞察分析
        round_insights = await task(
            "insight-agent",
            query_results=query_results,
            question=current_question
        )
        
        # 5. 累积洞察
        accumulated_insights.append({
            "round": current_round,
            "question": current_question,
            "insights": round_insights
        })
        
        # 6. 评估是否重规划
        replan_decision = await task(
            "replanner-agent",
            original_question=question,
            accumulated_insights=accumulated_insights,
            current_round=current_round,
            max_rounds=max_rounds
        )
        
        # 7. 判断是否继续
        if not replan_decision.need_replan:
            break
        
        current_round += 1
    
    # 8. 生成最终报告
    final_report = synthesize_final_report(accumulated_insights)
    return final_report
```

## ✅ 总结

### 三个概念的关系

```
┌─────────────────────────────────────────────────────────────┐
│                      完整分析流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  执行查询 → 返回数据                                         │
│      ↓                                                       │
│  渐进式洞察分析（针对单个查询结果）                          │
│      ├─ 分块分析                                             │
│      ├─ 累积洞察                                             │
│      └─ 智能早停                                             │
│      ↓                                                       │
│  合成本轮洞察                                                │
│      ↓                                                       │
│  Replanner 评估（基于累积洞察）                              │
│      ├─ 问题是否充分回答？                                   │
│      ├─ 需要什么新数据？                                     │
│      └─ 决定是否重规划                                       │
│      ↓                                                       │
│  如果需要重规划：                                            │
│      ├─ 生成新问题                                           │
│      ├─ 重新生成查询计划                                     │
│      ├─ 执行新查询（利用缓存）                               │
│      └─ 回到"渐进式洞察分析"                                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 关键点

1. **洞察的数据来源**：查询结果
2. **重规划的本质**：重新生成查询计划，执行新查询
3. **Replanner 的输入**：累积洞察（所有轮次）
4. **Replanner 的输出**：是否重规划 + 新问题
5. **缓存的作用**：重规划时避免重复查询

**这才是正确的集成方案！** ✅
