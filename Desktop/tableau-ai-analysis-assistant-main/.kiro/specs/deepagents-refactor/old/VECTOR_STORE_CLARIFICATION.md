# 向量存储澄清文档

## 🎯 核心问题

**向量数据库存储什么？**

---

## ❌ 错误理解

**误解**：把数据表的所有数据都放进向量数据库

```
错误示例：
┌─────────────────────────────────────┐
│      向量数据库（错误）              │
├─────────────────────────────────────┤
│ 销售数据（1000万行）                │
│ - 订单ID: 001, 地区: 华东, 销售额: 1000 │
│ - 订单ID: 002, 地区: 华北, 销售额: 500  │
│ - ...                                │
│ - 订单ID: 10000000, ...             │
└─────────────────────────────────────┘
```

**问题**：
- ❌ 数据量巨大（1000万行 × 10列 = 1亿个值）
- ❌ 成本极高（向量化成本 + 存储成本）
- ❌ 没有意义（数据分析不需要向量化所有数据）

---

## ✅ 正确理解

**正确**：只存储 Tableau 元数据（字段信息）

```
正确示例：
┌─────────────────────────────────────┐
│      向量数据库（正确）              │
├─────────────────────────────────────┤
│ 字段元数据（约 1000 个字段）         │
│                                      │
│ 字段 1:                              │
│ - 字段名: [Sales].[Sales Amount]    │
│ - 显示名: 销售额                     │
│ - 类型: measure                      │
│ - 数据类型: REAL                     │
│ - 描述: 销售金额总和                 │
│ - 示例值: [1000000, 500000, 250000] │
│                                      │
│ 字段 2:                              │
│ - 字段名: [Geography].[Region]      │
│ - 显示名: 地区                       │
│ - 类型: dimension                    │
│ - 数据类型: STRING                   │
│ - 描述: 销售地区                     │
│ - 示例值: ["华东", "华北", "华南"]  │
│                                      │
│ ... (约 1000 个字段)                 │
└─────────────────────────────────────┘
```

---

## 📊 数据量对比

| 项目 | 错误理解 | 正确理解 |
|------|---------|---------|
| **存储内容** | 数据表的所有数据 | 字段元数据 |
| **数据量** | 1000万行 × 10列 = 1亿个值 | 1000 个字段 |
| **向量数** | 1亿个向量 | 1000 个向量 |
| **存储空间** | 约 600GB（1亿 × 1536维 × 4字节） | 约 6MB（1000 × 1536维 × 4字节） |
| **构建成本** | 约 $13,000（text-embedding-3-large） | 约 $0.013（text-embedding-3-large） |
| **查询速度** | 极慢（需要搜索 1亿个向量） | 极快（只搜索 1000 个向量） |

**结论**：正确理解的数据量是错误理解的 **1/100,000**！

## 🎯 向量存储的用途

### 用途：语义字段映射

**场景**：用户使用业务术语，系统自动映射到技术字段名

**流程**：
```
用户输入："销售额"
  ↓
向量化查询："销售额"
  ↓
向量检索（在 1000 个字段中搜索）
  ↓
找到候选字段：
  1. [Sales].[Sales Amount] (相似度: 0.95)
  2. [Revenue].[Total Revenue] (相似度: 0.78)
  3. [Sales].[Sales Count] (相似度: 0.65)
  ↓
LLM 语义判断
  ↓
最佳匹配：[Sales].[Sales Amount]
```

### 不是用途：数据分析

**错误理解**：在向量数据库中搜索销售数据

```
❌ 错误流程：
用户问："华东地区的销售额"
  ↓
向量化查询："华东地区的销售额"
  ↓
向量检索（在 1000万行数据中搜索）
  ↓
找到相关数据行
  ↓
返回结果
```

**正确流程**：
```
✅ 正确流程：
用户问："华东地区的销售额"
  ↓
Understanding Agent：提取实体
  - 维度：地区
  - 度量：销售额
  - 过滤：华东
  ↓
Semantic Mapping：映射字段
  - "地区" → [Geography].[Region]
  - "销售额" → [Sales].[Sales Amount]
  ↓
Planning Agent：生成查询
  - SELECT SUM([Sales].[Sales Amount])
  - WHERE [Geography].[Region] = '华东'
  ↓
Query Executor：执行查询（在 Tableau 数据源）
  ↓
返回结果：华东地区销售额 = 500万
```

## 📝 存储内容详解

### 字段元数据包含什么？

```python
field_metadata = {
    # 基础信息
    "field_name": "[Sales].[Sales Amount]",  # 技术字段名
    "display_name": "销售额",                 # 显示名
    "field_role": "measure",                  # 字段角色
    "data_type": "REAL",                      # 数据类型
    
    # 业务信息
    "description": "销售金额总和",            # 描述
    "folder": "Sales",                        # 文件夹
    "aggregation": "SUM",                     # 默认聚合
    
    # 示例数据（用于理解数据规模）
    "sample_values": [1000000, 500000, 250000],
    
    # 关联信息
    "related_fields": [
        "[Sales].[Sales Count]",
        "[Sales].[Discount]"
    ],
    
    # 同义词（可选）
    "synonyms": ["收入", "营收", "Sales"],
    
    # 使用频率（可选）
    "usage_frequency": 150
}
```

### 向量化什么？

**向量化内容**：字段的富文本描述

```python
def build_field_text(field):
    """构建用于向量化的富文本描述"""
    parts = []
    
    # 1. 字段名（多种形式）
    parts.append(f"字段名: {field['field_name']}")
    parts.append(f"显示名: {field['display_name']}")
    
    # 2. 类型信息
    parts.append(f"类型: {field['field_role']}")
    parts.append(f"数据类型: {field['data_type']}")
    
    # 3. 业务描述
    if field.get('description'):
        parts.append(f"描述: {field['description']}")
    
    # 4. 文件夹/分类
    if field.get('folder'):
        parts.append(f"分类: {field['folder']}")
    
    # 5. 示例值
    if field.get('sample_values'):
        samples = ', '.join(str(v) for v in field['sample_values'][:3])
        parts.append(f"示例值: {samples}")
    
    # 6. 同义词
    if field.get('synonyms'):
        synonyms = ', '.join(field['synonyms'])
        parts.append(f"同义词: {synonyms}")
    
    return "\n".join(parts)

# 示例输出：
"""
字段名: [Sales].[Sales Amount]
显示名: 销售额
类型: measure
数据类型: REAL
描述: 销售金额总和
分类: Sales
示例值: 1000000, 500000, 250000
同义词: 收入, 营收, Sales
"""
```

## ✅ 总结

### 核心要点

1. ✅ **只存储元数据**：字段信息，不是数据表的所有数据
2. ✅ **数据量小**：约 1000 个字段，不是 1000万行数据
3. ✅ **成本极低**：约 $0.013 或 ¥0（BCE），不是 $13,000
4. ✅ **速度极快**：约 1 毫秒，不是 10 秒
5. ✅ **用途明确**：语义字段映射，不是数据分析

### 数据流

```
Tableau 数据源
  ├─ 元数据（字段信息）→ 向量数据库 ✅
  └─ 数据表（销售数据）→ Tableau 数据源 ✅
                        ↓
                    VizQL 查询
```

### 关键区别

| 项目 | 元数据 | 数据表 |
|------|--------|--------|
| **内容** | 字段信息 | 销售数据 |
| **数量** | 约 1000 个 | 约 1000万行 |
| **存储位置** | 向量数据库 | Tableau 数据源 |
| **用途** | 字段映射 | 数据分析 |
| **查询方式** | 向量检索 | VizQL 查询 |
